## 快速入门

### 首次登录

1. 打开浏览器访问 SwanEVAL 前端地址（默认 `http://localhost:3000`）
2. 系统检测到无用户时，会显示注册表单
3. 填写用户名、邮箱和密码完成注册
4. 首个注册用户自动获得管理员（admin）角色

### 5 分钟上手评测

完成以下步骤即可运行第一次评测：

```
① 注册模型 → ② 导入数据集 → ③ 选择评测标准 → ④ 创建任务 → ⑤ 查看结果
```

**① 注册模型**：进入「模型」页面 → 点击「添加模型」→ 选择「API 模型」标签 → 填写端点地址和 API 密钥

**② 导入数据集**：进入「数据集」页面 → 点击「新建」→ 在预设列表中点选 GSM8K → 等待导入完成

**③ 选择标准**：进入「评估标准」→ 在预设面板中点选「精确匹配」

**④ 创建任务**：进入「评测任务」→ 点击「新建任务」→ 依次选择模型、数据集、标准 → 提交

**⑤ 查看结果**：任务完成后，进入「结果分析」查看评分排行榜和详细报告

## 模型管理

### 添加模型

点击「添加模型」后，选择模型类型：

#### API 模型（调用外部 API）

适用于 OpenAI、Anthropic、自建推理服务等已有 API 端点的模型。

| 字段 | 必填 | 说明 |
|------|------|------|
| 显示名称 | ✅ | 模型在平台中的名称 |
| 提供商 | ✅ | 如 openai、anthropic |
| API 协议 | | OpenAI 或 Anthropic 格式 |
| 端点 URL | ✅ | API 地址，如 `https://api.openai.com/v1/chat/completions` |
| 模型 ID | | 实际调用的模型名，如 `gpt-4o` |
| API 密钥 | | 认证密钥 |

#### 自托管模型（部署到集群）

适用于 HuggingFace 或 ModelScope 上的开源模型，将通过 vLLM 自动部署到 K8s 集群。

| 字段 | 必填 | 说明 |
|------|------|------|
| 模型来源 | ✅ | HuggingFace 或 ModelScope |
| 模型 ID | ✅ | 仓库 ID，如 `Qwen/Qwen2.5-7B-Instruct` |
| 显示名称 | | 留空则自动取模型 ID 末段 |
| 计算集群 | ✅ | 选择已注册的 K8s 集群 |
| GPU 数量 | | 默认 1，设为 0 可 CPU 模式运行 |
| 内存 (GB) | | 默认 40 |

点击「注册并部署」后，系统将：
1. 在平台注册模型记录
2. 在选定集群上自动部署 vLLM 推理服务
3. 部署完成后模型状态变为「运行中」

> **Token 配置**：访问私有模型（如 Llama）需要在「账户设置」中配置 HuggingFace Token。表单底部会显示 Token 配置状态。

### 模型详情面板

点击模型列表中的任意模型，右侧打开详情面板：

- **基本信息**：名称、提供商、端点、模型 ID 等（可内联编辑）
- **连接状态**：绿色/红色/灰色圆点表示在线/离线/未测试
- **测试连接**：发送最小请求验证模型可用性
- **Playground**：展开后可直接输入 Prompt 测试模型回复，显示延迟和 Token 数
- **部署状态**：自托管模型显示 vLLM 部署状态（运行中/部署中/已停止/失败）
- **停止部署**：运行中的模型可一键停止 vLLM 部署释放资源

### 批量操作

勾选多个模型后，底部浮动操作栏可批量删除。

## 数据集管理

### 导入数据集

点击「新建」后，有以下导入方式：

#### 在线导入

从 HuggingFace 或 ModelScope 下载数据集：

1. 选择数据源（HuggingFace / ModelScope）
2. 输入 Dataset ID（如 `openai/gsm8k`）或完整 URL
3. 可选填子集（Subset）和数据拆分（Split）
4. 点击「导入数据集」

导入过程通过 SSE 实时推送进度，顶部进度指示器显示当前状态。

#### 文件上传

支持拖放或点击选择文件，格式支持：JSONL、CSV、JSON、Parquet、Excel

#### 服务器路径

直接注册服务器本地文件路径，无需复制文件（挂载模式）。

#### 预设数据集

侧边面板展示 18 个预置评测基准数据集，按类别分组。点击即可一键导入。详见 [附录](#附录预置资源清单)。

### 数据集详情

点击数据集名称打开详情面板，包含三个标签页：

#### 信息标签

- 名称、描述、来源类型、格式、标签
- 文件大小、数据行数、版本号
- 创建时间、来源 URI
- 下载状态（预设数据集）
- 订阅更新设置（HuggingFace 数据集）

#### 统计标签

自动分析数据集各字段的统计信息：
- 总览：行数、列数、文件大小
- 逐字段：数据类型、空值率（进度条）、唯一值数量、平均文本长度、高频值 TOP 5

#### 版本标签

- 版本历史列表（最新在前）
- 每个版本：行数、文件大小、变更说明、创建时间
- 当前版本标记
- 订阅数据集的同步日志（触发方式、状态、耗时、错误信息）

### 数据预览

点击「预览」按钮打开全屏对话框，显示前 50 行数据：
- 表头显示所有列名
- 长文本自动截断，点击可展开查看完整内容
- 支持翻页浏览
- 单行详情视图

### 自动更新订阅

HuggingFace 数据集支持自动更新：
1. 在详情面板中开启「订阅更新」
2. 设置检查间隔（默认 24 小时）
3. 系统后台定期检查远程仓库变更
4. 有更新时自动下载并创建新版本

## 评测标准管理

### 四类评测标准

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **预设指标** | 内置评分算法 | 通用评测 |
| **正则表达式** | 正则匹配模型输出 | 格式验证、信息抽取 |
| **沙箱执行** | 在隔离环境中运行代码 | 代码生成评测 |
| **LLM 评判** | 另一个大模型打分 | 主观质量评估 |

### 创建评测标准

#### 预设指标

选择一个内置评分算法，无需额外配置：

| 指标 | 算法 | 输出 |
|------|------|------|
| 精确匹配 | 去空白后严格比较 | 0 或 1 |
| 包含匹配 | 输出是否包含预期字符串 | 0 或 1 |
| 数值接近 | 提取数值，容差内比较 | 0 或 1 |
| BLEU | N-gram 重叠度 | 0 ~ 1 |
| ROUGE-L | 最长公共子序列 F 值 | 0 ~ 1 |
| F1 分数 | Token 级精确率与召回率 | 0 ~ 1 |
| 余弦相似度 | 字符 N-gram 向量相似度 | 0 ~ 1 |

#### 正则表达式

配置正则模式匹配模型输出：
- 支持完整 Python 正则语法
- 输入框提供实时语法高亮和校验
- 匹配成功返回 1.0，否则 0.0

#### 沙箱执行（代码评测）

两种模式：

**Pass@k 代码评测**：
1. 模型生成的代码写入临时文件
2. 预期输出中的测试断言追加到末尾
3. 在隔离子进程中执行
4. 退出码 0 = 通过 (1.0)，否则 = 失败 (0.0)

**自定义评估脚本**：
- 指定服务器上的 Python 脚本路径
- 脚本需提供 `evaluate(expected, actual) -> float` 函数
- 返回 0-1 之间的分数

#### LLM 评判

使用另一个大语言模型作为评分员：
1. 选择评判模型（从已注册模型中选择）
2. 编写系统提示词（或使用模板）
3. 可配置多维度评分（准确性、完整性、流畅性等）

提供三个提示词模板：
- 通用评判
- ELO 对比评判
- 流畅度评估

### 测试评测标准

创建后可在详情面板中进行干运行测试：
1. 输入预期输出和实际输出
2. 点击「测试」
3. 查看评分结果

## 评测任务

### 创建评测任务

点击「新建任务」进入 5 步向导：

#### 第 1 步：选择模型

从下拉列表中选择一个已注册的模型。

#### 第 2 步：数据集与评测标准

- **选择数据集**：点击多选，已选数据集高亮显示
- **字段映射**：每个数据集自动检测 Prompt 和 Expected 字段，可手动调整
- **选择评测标准**：点击多选

#### 第 3 步：参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 任务名称 | 必填 | 可点击骰子图标随机生成 |
| 温度 | 0.7 | 模型采样温度 |
| 最大 Token | 2048 | 单次生成上限 |
| 数据量限制 | 不限 | 仅评测前 N 条数据 |
| 重复次数 | 1 | 大于 1 时进行稳定性测试 |
| 种子策略 | 固定 | 固定种子或随机种子 |

#### 第 4 步：运行环境

| 参数 | 说明 |
|------|------|
| 执行后端 | 外部 API / 本地 Worker / K8s vLLM |
| 计算集群 | 选择 K8s/vLLM 后端时必选 |
| GPU 数量 | K8s 模式下的 GPU 需求 |
| 显存 (GB) | K8s 模式下的内存限制 |
| GPU 编号 | 本地模式下指定 CUDA 设备 |
| 环境变量 | JSON 格式的额外环境变量 |

#### 第 5 步：确认提交

显示所有配置的汇总信息，支持查看 JSON 配置。点击「提交任务」。

### 任务执行

任务提交后进入队列，由 Worker 进程执行：

1. **排队中**（pending）→ 等待 Worker 拾取
2. **运行中**（running）→ 逐条发送 Prompt 并评分
3. **完成**（completed）/ **失败**（failed）

#### 进度监控

任务详情页面实时显示：
- 总 Prompt 数 / 已完成数 / 预计剩余时间
- 每个子任务的独立进度条
- 实时刷新（3 秒间隔）
- 执行后端和 Worker ID

#### 操作

| 操作 | 条件 | 效果 |
|------|------|------|
| 暂停 | 运行中 | 暂停执行，保留进度 |
| 继续 | 已暂停/失败 | 从最后检查点恢复 |
| 取消 | 运行中/暂停 | 终止任务 |
| 重启 | 失败/已取消 | 清除结果，重新执行 |

### 断点续测

任务支持真正的断点恢复：
- 每个子任务记录已完成的 Prompt 索引
- 恢复时跳过已完成的 Prompt
- 历史结果不会被覆盖
- 已完成的子任务不会重复执行

### 稳定性测试

设置重复次数 > 1 时，系统创建多个子任务并行执行，完成后在结果分析中可查看：
- 均值、标准差、方差
- 95% 置信区间
- 每次运行的独立分数

## 结果分析与报告

进入「结果分析」页面，左侧导航栏提供以下视图：

### 排行榜

按模型和评测标准聚合的排名表：
- 支持按评测标准筛选
- 列：排名、模型名、标准名、平均分、评测数、平均延迟
- 合并本地评测结果与外部基准数据
- 支持导出 CSV

### 天梯榜

每个评测标准下的最佳模型：
- 区分本地评测和外部基准来源
- 显示来源徽标

### 对比图表

**柱状图**：多选模型 × 多选标准 → 分组柱状图

**雷达图**：单选模型 × 多选标准 → 多维能力雷达图

### 外部基准数据

导入闭源模型的公开评测数据用于对比：

**平台拉取**：从 Open LLM Leaderboard 自动拉取（支持按模型名筛选）

**手动导入**：粘贴 JSON 格式的评测数据

### 报告生成

选择已完成的任务，生成以下类型的报告：

| 报告类型 | 内容 |
|----------|------|
| 性能报告 | 各标准评分、总体分数、样本数 |
| 安全报告 | 错误率、风险等级、错误案例 TOP 20 |
| 成本报告 | 延迟统计、Token 消耗、吞吐量 |
| 价值报告 | 综合性能与成本的性价比指数 |

报告支持导出为：**PDF、HTML、DOCX、CSV**

生成的报告会持久化保存，可在报告列表中随时查看。

### 错误分析

错误分析区分两类错误：
- **模型回答错误**：模型给出了错误的答案（is_valid=true, score < 1.0）
- **运行执行错误**：基础设施故障（模型超时、评测器崩溃等）

## 计算资源管理

### 添加集群

1. 进入「计算资源」页面
2. 点击「添加」
3. 填写：
   - **集群名称**：如 "实验室 GPU 集群"
   - **命名空间**：默认 `default`
   - **Kubeconfig**：粘贴集群的 kubeconfig YAML 内容
   - **vLLM 镜像源**：可选阿里云/华为云镜像加速

系统自动验证连接并在后台探测集群资源。

### 获取 Kubeconfig

```bash
# k3s
sudo cat /etc/rancher/k3s/k3s.yaml

# 通用方式
kubectl config view --raw
```

### 集群详情

点击集群名称查看详情：

**基本信息**：API Server 地址、命名空间、vLLM 镜像源、创建时间

**资源概览**：GPU 数量/类型、CPU 核数、内存总量

**节点列表**：每个节点的 GPU/CPU/内存和状态

**活跃部署**：当前在集群上运行的 vLLM 模型部署

### GPU 支持

详情面板的「GPU 支持」区域显示实时 GPU 状态：

- **GPU 节点数量**和节点名称
- **Device Plugin** 运行状态
- **GPU Operator** 运行状态

如果未检测到 GPU 支持，可选择安装：

| 方式 | 说明 | 前提 |
|------|------|------|
| Device Plugin | 轻量方案，仅安装设备插件 | 节点已安装 NVIDIA 驱动，后端需要 kubectl |
| GPU Operator | 完整方案，自动管理驱动和插件 | 后端需要 [Helm CLI](https://helm.sh/docs/intro/install/) |

> **注意**：kubectl 和 Helm 需要安装在运行 SwanEVAL 后端的机器上，不是集群节点上。Docker 部署时已内置。

### GPU 显存估算

```
所需显存 ≈ 模型参数量 × 2 字节 × 1.2（开销系数）

7B 模型  → ~17 GB → 1 张 24GB 显卡
14B 模型 → ~34 GB → 2 张 24GB 显卡
72B 模型 → ~173 GB → 4 张 80GB A100
```

多卡部署时系统自动配置 tensor parallelism。

## 用户与权限管理

### 角色体系

| 角色 | 说明 | 关键权限 |
|------|------|---------|
| **管理员** | 系统完全控制 | 所有操作 + 用户管理 |
| **数据管理员** | 管理评测资源 | 数据集/标准/模型增删改，任务查看 |
| **工程师** | 执行评测任务 | 任务全流程管理，数据集/模型只读 |
| **观察者** | 只读访问 | 查看所有模块，不可修改 |

### 权限组

管理员可创建权限组，将细粒度权限分配给用户组：

**可用权限**（19 项）：
- 数据集：读取、写入、下载
- 模型：读取、写入
- 评测标准：读取、写入
- 任务：读取、创建、管理
- 结果：读取
- 报告：读取、生成、导出
- 集群：读取、管理
- 管理：用户管理、权限组管理、ACL 管理

### 资源级权限（ACL）

除角色和权限组外，还支持对单个资源设置访问控制：

| 访问级别 | 说明 |
|----------|------|
| 查看 | 仅查看 |
| 评测 | 可用于评测 |
| 下载 | 可下载 |
| 编辑 | 可修改 |
| 管理 | 完全控制 |

### 报告权限

报告支持独立于任务的可见范围控制：
- **仅创建者**：仅报告创建者可见
- **团队**：所有认证用户可见
- **公开**：所有用户可见
- **白名单**：指定用户 ID 可见

## 常见问题

### 任务一直在排队不执行？

确认 Worker 进程正在运行。默认配置下 Worker 内嵌在 API 进程中（`EMBEDDED_WORKER=true`）。如果设置了 `EMBEDDED_WORKER=false`，需要单独启动 Worker：

```bash
cd backend
uv run python -m app.worker
```

### 数据集导入失败？

- 检查网络连接（HuggingFace 在中国可能需要代理）
- 私有数据集需要在「账户设置」中配置 Token
- 查看后端日志获取详细错误信息

### 模型测试连接失败？

- 确认端点 URL 格式正确（包含 `/v1/chat/completions`）
- 确认 API 密钥有效
- Anthropic 模型需选择 `anthropic` 协议格式

### 集群探测显示 0 GPU？

- 确认节点已安装 NVIDIA 驱动（`nvidia-smi` 可用）
- 在集群详情的 GPU 支持区域安装 Device Plugin 或 GPU Operator
- 安装后等待 30-60 秒再刷新

### 如何进行多模型对比？

1. 注册多个模型
2. 对每个模型创建相同数据集 + 标准的评测任务
3. 在「结果分析」→「排行榜」中查看综合排名
4. 在「对比」标签页中生成柱状图或雷达图
